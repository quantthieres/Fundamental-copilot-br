# Forecast Cache

Derived analytical caches generated from precomputed CVM data.

## Structure

```
forecast-cache/
  time-series/         <TICKER>.json  — Normalized per-metric financial time series (quarterly)
  baseline-forecasts/  <TICKER>.json  — Precomputed baseline model forecasts and backtest results
```

This directory contains **derived analytical caches** — not raw CVM data.
Each file is generated from the CVM quarterly cache at `src/data/cvm-cache/quarterly/`.

## Normalized time-series cache

The time-series cache transforms raw quarterly financial records into clean,
per-metric time series ready for forecasting models.

Generated by:

```bash
npm run time-series:precompute
```

Each `time-series/<TICKER>.json` file contains:

- One `FinancialTimeSeries` per metric (revenue, EBIT, margins, YoY growth rates, etc.)
- Quality metadata per series (observations, missing count, outlier flags)
- Points sorted chronologically with source annotations
- Warnings for quality issues detected during normalization

## Baseline forecast cache

The baseline-forecasts cache contains precomputed quantitative projections of
company fundamentals using simple, auditable TypeScript models (no ML, no Python).

Generated by:

```bash
npm run forecast:precompute:baseline
```

Each `baseline-forecasts/<TICKER>.json` file contains:

- Forecasts for up to 8 metrics: revenue, EBIT, net income, operating cash flow,
  free cash flow, EBIT margin, net margin, free-cash-flow margin.
- Walk-forward backtest results for all eligible models (naive, seasonal_naive,
  moving_average_4q, linear_trend) with MAE, RMSE, MAPE, sMAPE, WAPE.
- Best-model selection per metric (lowest WAPE, fallback to sMAPE).
- Forecast points with heuristic uncertainty bands (yhat, yhatLower, yhatUpper).
- Per-metric warnings about data quality, partial years, and interval interpretation.
- `quality: ForecastQualityDiagnostic` embedded per metric: reliability level (high/medium/low/insufficient),
  score (0–100), Portuguese-language reasons, and quality warnings. This is a diagnostic/explainability
  layer and does not constitute investment advice.

**These are simple quantitative baselines, not investment recommendations.**
No price targets, DCF outputs, fair value estimates, or recommendation signals
are produced. Forecasts project financial/fundamental metrics only.

## What to commit

- **Commit**: compact normalized JSON files in `time-series/` and `baseline-forecasts/`.
- **Do not commit**: raw CVM ZIP/CSV files — those are downloaded at generation time only.
- **Do not commit**: model artifacts, fitted model weights, or Python intermediate files.
  Only the output JSON is committed. No ML dependencies are required.
