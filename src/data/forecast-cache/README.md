# Forecast Cache

Derived analytical caches generated from precomputed CVM data.

## Structure

```
forecast-cache/
  time-series/  <TICKER>.json   — Normalized per-metric financial time series (quarterly)
```

This directory contains **derived analytical caches** — not raw CVM data.
Each file is generated from the CVM quarterly cache at `src/data/cvm-cache/quarterly/`.

## Normalized time-series cache

The time-series cache transforms raw quarterly financial records into clean,
per-metric time series ready for future forecasting models.

Generated by:

```bash
npm run time-series:precompute
```

Each `time-series/<TICKER>.json` file contains:

- One `FinancialTimeSeries` per metric (revenue, EBIT, margins, YoY growth rates, etc.)
- Quality metadata per series (observations, missing count, outlier flags)
- Points sorted chronologically with source annotations
- Warnings for quality issues detected during normalization

This is infrastructure for the **future Forecast Layer** — no forecasting models
are implemented yet. The cache schema is designed to feed baseline and statistical
forecasting models in a later phase.

## What to commit

- **Commit**: compact normalized JSON files in `time-series/`.
- **Do not commit**: raw CVM ZIP/CSV files — those are downloaded at generation time only.
- **Do not commit**: model artifacts, fitted model weights, or Python intermediate files.
  Only the output JSON is committed.

## What is NOT here yet

- Forecast outputs (yhat, confidence intervals, model metadata)
- Model comparison tables
- Backtesting results

These belong to a later Forecast Layer phase after baseline models are validated.
